## 二、统一架构规范

### 2.1 分层规范

| 层级 | 命名规范 | 职责 | 禁止事项 |
|:---|:---|:---|:---|
| **Entity** | `XxxEntity` | 与数据库表一一对应，仅含字段和注解 | 禁止包含业务逻辑、DTO、VO |
| **DTO** | `XxxDTO` / `CreateXxxDTO` / `UpdateXxxDTO` | 接口入参，配合 `@Valid` 校验 | 禁止包含实体无关字段、禁止直接返回给前端 |
| **VO** | `XxxVO` / `XxxDetailVO` / `XxxListVO` | 接口出参，控制返回字段 | 禁止包含密码等敏感字段 |
| **Mapper** | `XxxMapper` | 继承 `BaseMapper`，仅负责数据访问 | 禁止写业务逻辑、禁止参数校验 |
| **Service** | `XxxService`（接口）+ `XxxServiceImpl`（实现） | 核心业务逻辑、事务控制、数据编排 | 禁止直接操作 HttpServletRequest/Response |
| **Controller** | `XxxController` | 接收请求、参数校验、调用 Service、返回 ApiResult | **禁止写业务逻辑**、禁止直接调用 Mapper |
| **Convert** | `XxxConvert`（MapStruct 接口） | Entity DTO VO 的对象转换 | 禁止手写 set/get 转换 |

### 2.2 对象转换规范（MapStruct）

```java
@Mapper(componentModel = "spring")
public interface WorkerConvert {
    WorkerConvert INSTANCE = Mappers.getMapper(WorkerConvert.class);
    WorkerEntity toEntity(CreateWorkerDTO dto);
    void updateEntity(@MappingTarget WorkerEntity entity, UpdateWorkerDTO dto);
    WorkerDetailVO toDetailVO(WorkerEntity entity);
    List<WorkerListVO> toListVO(List<WorkerEntity> list);
}
```

### 4.1 异常体系

```

    ├── RuntimeException
    │       ├── BusinessException（已知业务错误）
    │       └── 其他运行时异常（未知错误）
```

|:---|:---|:---|

| `BindException` | 提取字段绑定错误 | 400 |

| `AccessDeniedException` | 权限不足 | 403 |

## 五、登录体系设计（重点）

### 5.1 双身份认证模型

系统存在两套用户表，但统一封装为 `LoginUser`：

```
admins 表 (管理员)          drivers 表 (司机)
  username                    phone
  password                    password
  real_name                   real_name
  is_active                   is_active
                              password_changed
         │                         │
         └───────────┬─────────────┘
                     ▼
            UserDetailsServiceImpl
            loadUserByUsername()
            先查 admin，再查 driver
                     │
                     ▼
                 LoginUser
              userId / username
              realName / role
              passwordChanged
```

### 5.2 管理员登录流程

```
POST /api/admin/auth/login
    │
    ▼
UserDetailsService.loadUserByUsername(username)
    │
    ▼
AuthenticationManager.authenticate()
    │
    ▼
BCryptPasswordEncoder.matches(rawPassword, encodedPassword)
    │
    ▼
生成 JWT Token → 存入 Redis → 返回前端
```

### 5.3 司机登录流程（两阶段）

**第一阶段：账号密码登录**

```
POST /api/driver/auth/login
    │
    ▼
手机号 + 密码 认证（username=手机号）
    │
    ▼
校验 password_changed 字段
    ├── false → 返回状态码 1004（首次登录必须修改密码）
    └── true  → 生成 JWT → 存入 Redis → 返回前端
```

**第二阶段：微信绑定与快捷登录**

```
司机已登录状态下：
    │
    ▼
POST /api/driver/auth/bind-wechat
    参数：wx.login() 获取的 code
    │
    ▼
调用微信接口 code2session 换取 OpenID
    │
    ▼
将 OpenID 写入 drivers 表的 wx_openid 字段
    │
    ▼
后续登录：POST /api/driver/auth/wx-login
    通过 OpenID 查找司机 → 直接生成 JWT（无需密码）
```

### 5.4 JWT 认证流程图

```
请求到达
    │
    ▼
JwtAuthenticationFilter (OncePerRequestFilter)
    │
    ├── 1. 从 Header 提取 Token（Authorization: Bearer xxx）
    │
    ├── 2. 解析 Token 获取 userId + role
    │
    ├── 3. 到 Redis 校验 Token 是否仍有效
    │   key = token:admin:{userId} 或 token:driver:{userId}
    │
    ├── 4. 构建 LoginUser 对象
    │
    └── 5. 写入 SecurityContextHolder
                │
                ▼
        后续 @PreAuthorize 判断角色权限
                │
                ▼
        Controller / Service 通过 SecurityUtils 获取当前用户
```

## 七、审计字段设计

### 7.1 BaseEntity 设计

| 字段 | 类型 | 说明 | 填充策略 |
|:---|:---|:---|:---|
| id | BIGINT | 主键，自增 | — |
| createTime | DATETIME | 创建时间 | INSERT 自动填充 |
| updateTime | DATETIME | 更新时间 | INSERT + UPDATE 自动填充 |
| createBy | BIGINT | 创建人ID | INSERT 自动填充 |
| updateBy | BIGINT | 更新人ID | INSERT + UPDATE 自动填充 |
| deleted | INT | 逻辑删除标志 | INSERT 自动填充（默认 0）|
| version | INT | 乐观锁版本号 | INSERT 自动填充（默认 1）|

### 7.2 逻辑删除方案

- **实现方式**：MyBatis-Plus `@TableLogic` 注解
- **deleted = 0**：数据正常存在
- **deleted = 1**：数据已被逻辑删除
- **查询影响**：所有 `BaseMapper.selectXxx` 自动附加 `WHERE deleted = 0`
- **删除影响**：调用 `deleteById` 实际执行 `UPDATE ... SET deleted = 1`

## 八、日志系统设计

### 8.1 @OperationLog 注解设计

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface OperationLog {
    String module();           // BATCH / WORKER / DRIVER / WAGE / PROJECT / SYSTEM / AUTH
    String type();             // CREATE / UPDATE / DELETE / REVIEW / SETTLE / WITHDRAW / LOGIN
    String desc() default "";
    boolean recordParams() default true;
    boolean recordResult() default true;
}
```

### 8.2 AOP 实现方式

采用 **@Around 环绕通知**：

```
方法调用前
    ├── 记录开始时间
    ├── 获取当前登录用户（SecurityContextHolder）
    ├── 获取 HTTP 请求（RequestContextHolder）
    └── 读取 @OperationLog 注解信息

执行目标方法
    ├── 成功 → 记录成功状态和返回值
    └── 异常 → 记录失败状态和异常信息

方法调用后（finally）
    ├── 计算执行耗时
    ├── 组装 LogEntity
    └── 插入 operation_logs 表
```

### 8.3 需要记录的关键操作

| 操作人 | 操作模块 | 操作类型 | 场景 |
|:---|:---|:---|:---|
| 司机 | BATCH | CREATE | 提交考勤批次 |
| 司机 | BATCH | WITHDRAW | 撤回待审核批次 |
| 司机 | AUTH | UPDATE | 修改密码 |
| 管理员 | BATCH | REVIEW | 审核考勤批次 |
| 管理员 | WORKER | CREATE | 新增工人 |
| 管理员 | WORKER | UPDATE | 修改工人信息 |
| 管理员 | WORKER | DELETE | 删除工人 |
| 管理员 | DRIVER | CREATE | 新增司机 |
| 管理员 | DRIVER | UPDATE | 修改司机信息 |
| 管理员 | WAGE | SETTLE | 工资结清 |

### 8.4 性能优化建议

当前实现为**同步写入**数据库。如日志量增大，建议：
1. 改为**异步线程池**写入（`@Async`）
2. 或引入 **MQ** 削峰
3. 日志表按月份分区，定期归档

## 十、代码规范要求

### 10.1 强制规范

| 规范 | 要求 |
|:---|:---|
| **中文注释** | 所有类、所有方法、所有复杂逻辑块必须有中文注释 |
| **语义化命名** | 变量/方法/类名见名知意，禁止 `a`、`b`、`tmp1` |
| **Controller 职责** | **禁止写业务逻辑**，仅负责：接收请求 -> 参数校验 -> 调用 Service -> 返回 ApiResult |
| **Service 职责** | 负责业务逻辑、事务控制、权限校验、数据编排 |
| **Mapper 职责** | **只负责数据库访问**，禁止写业务逻辑 |
| **异常处理** | Service 层抛 `BusinessException`，禁止吞掉异常后返回 null |
| **对象转换** | Entity DTO VO 必须使用 MapStruct，禁止手写 BeanUtils.copyProperties |
| **空指针保护** | 所有可能为 null 的字段必须判空 |

### 10.2 代码示例（规范参考）

```java
/**
 * 审核考勤批次
 *
 * <p>审核通过后：</p>
 * <ol>
 *     <li>批次状态变为"已通过"</li>
 *     <li>为每个工人考勤记录计算并固化工资</li>
 *     <li>为司机生成当日考勤记录</li>
 * </ol>
 *
 * @param batchId 批次ID
 * @param dto     审核参数
 * @throws BusinessException 批次不存在或状态不允许审核时抛出
 */
@Transactional(rollbackFor = Exception.class)
@OperationLog(module = "BATCH", type = "REVIEW", desc = "审核考勤批次")
public void reviewBatch(Long batchId, BatchReviewDTO dto) {
    // 1. 校验批次是否存在且状态为"待审核"
    AttendanceBatchEntity batch = batchMapper.selectById(batchId);
    if (batch == null) {
        throw new BusinessException(ResultCodeEnum.DATA_NOT_FOUND, "批次不存在");
    }
    if (!BatchStatusEnum.PENDING.getCode().equals(batch.getStatus())) {
        throw new BusinessException(ResultCodeEnum.BATCH_STATUS_ERROR);
    }

    // 2. 校验所有工人考勤记录的项目是否已分配
    List<WorkerAttendanceRecordEntity> records = recordMapper.selectByBatchId(batchId);
    for (WorkerAttendanceRecordEntity record : records) {
        if (record.getProjectId() == null) {
            throw new BusinessException(ResultCodeEnum.PROJECT_REQUIRED,
                    "工人未分配项目，无法审核");
        }
    }

    // 3. 更新批次状态为"已通过"
    batch.setStatus(BatchStatusEnum.APPROVED.getCode());
    batch.setReviewTime(LocalDateTime.now());
    batch.setReviewerId(SecurityUtils.getCurrentUserId());
    batchMapper.updateById(batch);

    // 4. 计算并写入每个工人的当日工资
    for (WorkerAttendanceRecordEntity record : records) {
        calculateAndSaveWage(record);
    }

    // 5. 生成司机当日考勤记录
    createDriverAttendance(batch);
}
```
